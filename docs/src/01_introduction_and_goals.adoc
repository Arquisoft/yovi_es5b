== Introducción y objetivos

La empresa de desarrollo de juegos Micrati ha decidido desarrollar una plataforma web donde los usuarios puedan jugar diferentes variantes del juego Y, un juego estratégico de tablero existente. El objetivo inicial es ofrecer una experiencia entretenida e intuitiva que atraiga a nuevos usuarios y los mantenga comprometidos con el producto a largo plazo. La plataforma (YOVI) estará disponible en web, permitirá jugar contra diferentes estrategias de bots con niveles de dificultad seleccionables. El motor de reglas y verificación está implementado en Rust, mientras que el cliente web principal está desarrollado en TypeScript.

=== Resumen de requisitos

Los requisitos del proyecto Yovi son:

- Un usuario puede registrarse en el sistema y posteriormente iniciar sesión con credenciales seguras.
- Un usuario puede consultar su histórico de participación, visualizando el número total de juegos realizados y las estadísticas de partidas ganadas/perdidas.
- Un usuario puede iniciar y jugar a la versión clásica del juego Y enfrentándose a la máquina en un tablero de tamaño variable.
- Un usuario puede seleccionar qué estrategia algorítmica utilizará el bot rival (pudiendo elegir entre más de una) y seleccionar su nivel de dificultad.
- La aplicación tendrá un API documentada que permitirá a bots interactuar con ella.
- La API permitirá acceder y gestionar información de los usuarios y de las partidas
- La API permitirá que un bot pueda jugar contra la aplicación.

=== Objetivos de calidad

[options="header",cols="1,2"]
|===
|Objetivo de calidad|Descripción
|Usabilidad|El sistema debe ser intuitivo y ofrecer una excelente experiencia de juego, fomentando que los usuarios vuelvan a utilizar la aplicación. Se verificará mediante la realización de tests de usabilidad.
|Rendimiento|La aplicación web debe ser ágil, buscando que los tiempos de respuesta sean lo más cortos posible en las interacciones principales (ej. cálculo de movimiento del bot). Se verificará mediante métricas de observabilidad en tiempo real como Prometheus o Grafana.
|Mantenibilidad|El sistema debe seguir buenas prácticas arquitectónicas, tener un código limpio y una alta calidad de implementación.
|Disponibilidad|La aplicación debe estar desplegada, accesible a través de la web to el tiempo posible, además, deberá funcionar correctamente ante cualquier situación.
|Exactitud|El motor del juego en Rust debe validar los movimientos, calcular las coordenadas en el sistema de ejes (x,y,z) baricéntricos y detectar condiciones de victoria correctamente.
|===
