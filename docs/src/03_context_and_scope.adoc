== Contexto y alcance


ifdef::arc42help[]
[role="arc42help"]
****
.Contenido
El alcance y contexto del sistema - como lo sugiere el nombre - delimita al sistema (es decir, el alcance) de todos sus
socios de comunicación (Usuarios y sistemas vecinos, es decir, el contexto del sistema). Con ello se especifican las interfaces externas.

Si es necesario, diferenciar el contexto de negocio (Entradas y salidas específicas del dominio) del contexto técnico (canales, protocolos, hardware).

.Motivación
Las interfaces de dominio y las interfaces técnicas a los socios de comunicación son de los aspectos más críticos del sistema. Asegúrate de que los comprendes totalmente.

.Forma
Varias opciones:

* Diagramas de contexto
* Listas de socios de comunicación y sus interfaces.


.Más información

Ver https://docs.arc42.org/section-3/[Context and Scope] en la documentación de arc42.

****
endif::arc42help[]

Esta sección describe el entorno en el que opera el sistema YOVI y las interacciones con actores externos.
El sistema se presenta como una caja negra, mostrando únicamente sus relaciones con el exterior.

=== Contexto de negocio

ifdef::arc42help[]
[role="arc42help"]
****
.Contenido
La especificación de *todos* los socios de comunicación (usuarios, sistemas...) con explicaciones de las entradas y salidas específicas de dominio o interfaces.
Opcionalmente se pueden añadir formatos específicos de dominio o protocolos de comunicación.

.Motivación
Todas las partes interesadas deben entender qué datos se intercambian en el entorno del sistema.

.Forma
Todo tipo de diagramas que muestren el sistema como una caja negra y especifiquen las interfaces de dominio a los socios de comunicación.
Alternativamente (o adicionalmente) se puede usar una tabla.
El título de la tabla es el nombre de tu sistema y las tres columnas contienen el nombre del socio de comunicación, las entradas y las salidas.

****
endif::arc42help[]

**Diagrama de contexto de negocio**

[plantuml]
----
@startuml
left to right direction
skinparam rectangleStyle rounded
title YOVI - Diagrama de contexto de negocio
actor Jugador
rectangle "Sistema YOVI"
Jugador --> "Sistema YOVI" : Se registra y juega
"Sistema YOVI" --> Jugador : Muestra estado del juego y estadísticas
@enduml
----

El actor externo principal del sistema es el *jugador*, que puede registrarse y participar en partidas.
El sistema YOVI proporciona al jugador información sobre el estado del juego y diferentes estadísticas.

=== Contexto técnico

ifdef::arc42help[]
[role="arc42help"]
****
.Contenido
Las interfaces técnicas (canales y medios de transmisión) que enlazan al sistema con su entorno. Además, el mapeo de las entradas/salidas específicas del dominio a los canales, es decir, una explicación sobre qué entrada/salida utiliza qué canal.

.Motivación
Muchas partes interesadas toman decisiones arquitectónicas basadas en las interfaces técnicas entre el sistema y su contexto. Especialmente los diseñadores de infrastructura o hardware deciden estas interfaces técnicas.

.Forma
Por ejemplo, los diagramas UML de despliegue describiendo canales a sistemas vecinos, junto con una tabla que muestre las relaciones entre canales y entrada/salida.

****
endif::arc42help[]
